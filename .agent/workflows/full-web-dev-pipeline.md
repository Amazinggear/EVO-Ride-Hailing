# خط الإنتاج الكامل لمشاريع الويب (The Ultimate Web Dev Pipeline)

// turbo-all

هاد الـ Workflow بيشرح كيف بتستخدم الـ 22 مهارة (Skill) الموجودين عندك بالترتيب الصحيح لتبني أي مشروع، من مرحلة التفكير (Brainstorming) لحد ما يتسلم المشروع شغال بدون أخطاء.

**قاعدة هامة للذكاء الاصطناعي:** يجب تنفيذ كل خطوة والانتقال للي بعدها تلقائياً (Autonomously). عند الانتهاء من فاز (Phase)، اعطي تقرير مختصر للمستخدم وكمل للفاز اللي بعده بدون ما توقفه كل شوي بأسئلة. حافظ على تنسيق اللغتين (RTL/LTR) في تقريرك.

## المرحلة صفر: التخطيط والهوية (التأسيس)
قبل ما تلمس سطر كود، لازم تعرف إنت شو رح تبني.
1. **استلهام الأفكار والـ UI/UX**:
   - استخدم `/ui-ux-inspiration-scout` عشان تجيب أمثلة تغذية بصرية (Inspiration) للمشروع.
2. **العصف الذهني والتخطيط**:
   - استدعي `brainstorming-ideas` و `planning-execution` لكسر المشروع لمهام واضحة واعتماد الـ Architecture.
3. **هوية العلامة التجارية**:
   - استخدم `brand-identity` عشان تعتمد الألوان (Design Tokens) والخطوط وطريقة الكلام بالموقع. إذا المشروع إسلامي أو بيحتاج سحب محتوى، استخدم `reliable-content-scraper`.

---

## المرحلة الأولى: بناء الهيكل ونظام العمليات (The Core Engine)
بمجرد ما تبدأ تكتب كود Next.js أو React:
1. **تحديد الفئة السعرية (الأوركسترا)**:
   - ابدأ بطلب `tier-based-architecture`، هاد بيحدد إذا المشروع راح يكون:
     - فئة 1 (عادي وتجاري)
     - فئة 2 (Premium 2D)
     - فئة 3 (Hybrid 3D)
     - فئة 4 (Immersive WebGL)
2. **منهجية التطوير**:
   - خليك شغال على نظام `test-driven-development` عشان ما يتراكم عليك Technical Debt للمستقبل.

---

## المرحلة الثانية: الواجهات والحركات (The Visual Layer)
بناءً على الفئة اللي اخترتها بالخطوة السابقة، استدعي الـ Skills الجمالية:
1. **إذا المشروع 2D فخم**:
   - وظف `high-end-tech-ui` لعمل الـ Dark Mode وتأثيرات الـ Glassmorphism.
   - ركب `advanced-scroll-animations` عشان الحركات تطلع سلسلة (Lenis/Framer).
   - إذا عجبك أنيميشن بفيديو لزبون، استخدم `replicating-motion` عشان تنسخ الكود تبعه بالضبط.
2. **لو المشروع بحتاج صوت**:
   - استدعي `immersive-web-audio` لتعطي روح للموقع (Ambient & UI Hover Sounds).

---

## المرحلة الثالثة: عالم الشاشات الثلاثية الأبعاد (The 3D Realm)
*(فقط لو اخترت مشروع Tier 3 أو Tier 4)*:
1. **تجهيز الماتيريال**:
   - أول خطوة هون هي `3d-asset-pipeline` لضغط السيارت أو المنتجات (GLTF) لمقاس بيزبط عالويب (Draco).
2. **هندسة العالم 3D**:
   - إذا عرض منتج بسيط: استخدم `react-three-fiber-basics`.
   - إذا موقع متكامل والكاميرا بتطير فيه: ادخل بـ `webgl-environment-architect`.
   - إذا في تأثيرات سحرية (ميّ/تشويه/بارتكلز): اطلب `custom-glsl-shaders`.
3. **الدقة والصبر (Loading Screens)**:
   - مواقع الـ 3D بتطول لتحمل! إياك تنسى تطلب مهارة `webgl-creative-preloaders` عشان تركب شاشة تحميل فخمة محترمة بتعد النسبة المئوية.

---

## المرحلة الرابعة: الفحص، التنظيف، وتسليم المفتاح (QA & Delivery)
الفرن خلص، هسا صار وقت نلمع الكيكة ونتأكد إنها مش محروقة:
1. **تحسين الأداء (أهم خطوة بالـ 3D)**:
   - اضرب الكود بمهارة `3d-performance-optimizer` عشان تتأكد إنو المتصفح مش رح يعمل Crash ولا التلفون بيحمى (60FPS).
2. **الفحص النقدي للـ UI/UX**:
   - انده على `ui-ux-critic-scout`. هاد رح يفلي الموقع تفلاي، ويعطيك انتقادات قاسية إذا في زوايا مش مزبوطة أو ألوان بتعمي العين.
3. **حل الأخطاء ومراجعة الكود**:
   - اقمع أي مشكلة بـ `handling-errors` و `error-handling-patterns`.
   - اطلب `strict-code-reviewer` يلقي نظرة أخيرة عالـ Pull Request تبعك ليتأكد إنه مطابق لشروط الـ Brand.
